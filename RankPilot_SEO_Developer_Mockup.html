🛠 Developer Guide — Onboarding

What is "ranking"?

When someone types "plumber Sydney" into Google, Google shows a list of businesses. The position in that list (1st, 2nd, 3rd...) is the "ranking". Our job is to track and improve these rankings.

What is a "suburb"?

Australia is divided into geographic zones called suburbs (like neighbourhoods). A plumber in Sydney may want to appear in Google search for 50 different suburbs around them.

🛠 Developer Guide — Dashboard

Visibility Score (0-100)

A number we calculate ourselves — it is NOT from any API. Formula: for each suburb we track, if the business ranks #1-3 = full score, #4-10 = 60%, #11-20 = 30%, not ranking = 0. Weight each suburb by its monthly search volume. The weighted average × 100 = Visibility Score.

1
4 summary numbers

All from our own database — no API call needed at display time. We calculate and store these during the weekly scan job, then just read them here.

④ Near-Miss Keywords Alert

What is "Near-Miss"?

A keyword ranking on Google page 2 (positions 11-20). Studies show 95%+ of users never go past page 1. So position 14 = almost zero traffic. But it is CLOSE — one targeted action can move it to page 1.

4
Query for near-miss keywords

Find keywords where position BETWEEN 11 AND 20. Sort by search volume (higher volume = more valuable to fix). Show top 3.

keyword_rankings 
SELECT keyword, position, monthly_volume
FROM keyword_rankings
WHERE client_id = ?
  AND position BETWEEN 11 AND 20
ORDER BY monthly_volume DESC
LIMIT 3

🛠 Developer Guide — Maps Tracker

Google Maps "Local Pack"

When you search "plumber Sydney" on Google, you see a map with 3 businesses listed below it. That is the "Local Pack". Ranking #1 in the local pack = top of the map. This is the most valuable position for a local business. We track this position per suburb.

① Visibility Score Calculation

1
Calculate in backend — store in DB

Do NOT calculate in frontend. Run this calculation after every scan and store the result. The frontend just reads it.

-- Pseudocode for Visibility Score
FOR each suburb_result in this week's scan:
  vol_weight = suburb.monthly_volume / max_volume
  IF rank_position <= 3 THEN rank_weight = 1.0
  IF rank_position <= 10 THEN rank_weight = 0.6
  IF rank_position <= 20 THEN rank_weight = 0.3
  IF rank_position IS NULL THEN rank_weight = 0
  
  weighted_sum += rank_weight * vol_weight
  total_possible += vol_weight

visibility_score = ROUND(weighted_sum/total_possible * 100)

② Suburb Heat Map — Leaflet.js

2
Real map uses Leaflet.js with GeoJSON polygons

Each suburb is a geographic shape (polygon). We colour each polygon based on rank_position. Use Leaflet.js (free, open source) with CARTO Positron map tiles (also free).

No API cost — Leaflet is freesuburb_coordinates 
// Leaflet setup
const map = L.map('map').setView([-33.86, 151.20], 11);
L.tileLayer('https://{s}.basemaps.cartocdn.com/
  light_all/{z}/{x}/{y}{r}.png').addTo(map);

// For each suburb with result data:
const color = getColor(result.rank_position);
L.geoJSON(suburb.geojson_polygon, {
  style: { fillColor: color, fillOpacity: 0.6,
           weight: 1, color: color }
}).addTo(map);

function getColor(pos) {
  if (!pos) return '#EF4444';    // red = not ranking
  if (pos <= 3) return '#22C55E'; // green = top 3
  if (pos <= 10) return '#86EFAC';// light green = page 1
  return '#FCD34D';               // yellow = page 2
}
What is a GeoJSON polygon?

A GeoJSON polygon is a list of lat/lng coordinate pairs that draw the outline of a suburb on a map. We store these in the suburb_coordinates table, loaded from the Australian Bureau of Statistics (ABS) dataset.

🛠 Developer Guide — Content Engine

Why suburb-specific pages?

Google ranks individual web pages, not entire websites. A page titled "Plumber Burwood" will rank for searches like "plumber Burwood". Without this page, the business is invisible in Burwood regardless of how good their services are. We create one page per suburb they want to rank in.

What is Schema Markup?

Hidden code on the page that tells Google "this is a LocalBusiness located in Burwood that provides plumbing services". It is written in a format called JSON-LD. Google reads it to understand the page — humans never see it. It increases chances of appearing in rich results.

Simple DB counts

Count rows in content_pages table filtered by status. No API needed.

FROM content_pages WHERE client_id = ?

When you search for a business on Google, you see a panel on the right (or top on mobile) showing their name, address, phone, photos, reviews, and hours. That panel is the Google Business Profile. It is also what appears in Google Maps. Optimising this listing improves ranking in Google Maps results.

🛠 Developer Guide — Citation Auditor

What is NAP and why does it matter?

NAP = Name, Address, Phone. Google cross-references your business details across the internet. If your phone number appears differently on 8 different websites, Google sees inconsistency and TRUSTS your listing less — lowering your Maps ranking. Consistent NAP = Google trust signal.

What is a "citation"?

Any website that lists your business name + address + phone number. Yellow Pages, Yelp, hipages, TrueLocal — all of these are "citations". The more consistent and numerous your citations, the better your Google Maps ranking.

🛠 Developer Guide — Keyword Monitor

What is "organic search"?

Organic search results are the regular links on Google (NOT Google Maps, NOT ads). When someone searches "plumber Sydney" and scrolls past the map, they see a list of websites. That is organic search. A business on page 1 (positions 1-10) gets traffic. Page 2 (positions 11-20) gets almost none. Page 3+ = invisible.

What is a "keyword"?

A keyword is the phrase people type into Google. "plumber Parramatta" is a keyword. "blocked drain Sydney" is a keyword. "emergency plumber near me" is a keyword. We track where the client's website appears in search results for each keyword, every week.

② Near-Miss Detection Logic

2
Automatic detection + action queuing

After each weekly scan, run this check. If a keyword is between position 11-20 AND no content action is already queued for it, automatically create a job in the job_queue table.

-- Run after weekly keyword scan:
INSERT INTO job_queue (client_id, job_type, payload, run_at)
SELECT 
  client_id,
  'generate_content',
  json_build_object(
    'keyword', kw.keyword,
    'position', kw.position,
    'suburb', kw.suburb,
    'content_type', 
      CASE WHEN kw.is_suburb_kw THEN 'suburb_page'
           WHEN kw.is_question_kw THEN 'faq_block'
           ELSE 'service_page' END
  ),
  NOW() + INTERVAL '1 day'
FROM keyword_rankings kw
WHERE kw.client_id = ?
  AND kw.position BETWEEN 11 AND 20
  AND NOT EXISTS (
    SELECT 1 FROM job_queue jq 
    WHERE jq.client_id = kw.client_id 
    AND jq.payload->>'keyword' = kw.keyword
    AND jq.status = 'pending'
  )

③ Keyword Table Data Structure

3
Store weekly snapshots — never overwrite

Add a new row each week instead of updating. This lets us show trends (was position 22 last week, now 14 = moved up 8). Change = this_week - last_week.

keyword_rankings 
keyword_rankings:
  rank_id UUID PK
  client_id UUID FK
  keyword TEXT
  suburb TEXT        -- extracted from keyword
  position INT       -- NULL if not in top 100
  scanned_at DATE    -- week of scan (not datetime)
  monthly_volume INT -- from DataforSEO keyword data
  page_url TEXT      -- which client page ranks
  dataforseo_task_id TEXT

-- Get current + previous for change calculation:
SELECT 
  curr.keyword,
  curr.position AS this_week,
  prev.position AS last_week,
  (prev.position - curr.position) AS change
FROM keyword_rankings curr
LEFT JOIN keyword_rankings prev 
  ON curr.client_id = prev.client_id
  AND curr.keyword = prev.keyword
  AND prev.scanned_at = curr.scanned_at - INTERVAL '7 days'
WHERE curr.client_id = ?
  AND curr.scanned_at = CURRENT_DATE
ORDER BY curr.monthly_volume DESC
